1. 백테스트 레이아웃 개선
 - 상단, 중앙, 하단으로 나눠서 동일한 폭으로 배치를 하는데 상단은 백테스트 설정, 중앙은 백테스트 결과, 하단은 최근 매매로그를 표시해줘
 - 상단 백테스트 설정에서 조회 조건들이 두줄 높이로 배치되게 하고 제목, 입력란, 버튼 등이 정돈된 모습이면 좋겠어
 - 하단 최근 매매로그는 백테스트 설정에서 데이터 확인 및 다운로드 클릭하고나서 데이터가 준비되면 바로 그 데이터를 표시해줘
 - 최근 매매로그에 표시되어야 할 항목은 전략에서 판단해야할 정보가 포함되어야 함
 
 
2. 백테스트 실행 기능 개선
 - 백테스트 실행 버튼을 클릭하면 최근 매매로그 자료를 위에서 부터 아래로 순서대로 읽어 나가면서 매매판단을 하고 매매정보를 기록해 나가면 돼
 - 이때 진행 상황을 중앙 백테스트 결과 영역의 하단에 프로그레스바 형태로 진행률을 표시하면서 채워져 나가면 사용자가 이해하기가 쉬울거 같아
 - 모든 자료에 대한 매매결과가 나오면 최종적으로 중앙 백테스트 결과에 계산값을 표시하면 돼
 - 현재 방식은 가시적이지 않아서 제대로 백테스트가 진행되었는지 알 수가 없고, 매매가 한건도 이루어지지 않았는데 총 수익률을 -100% 로 표시하는 문제가 있어
 

3. 백테스트 로그 문제
 - anti_stock.log 파일에서 백테스트 로그를 보면 동일한 로그가 데이터 건수만큼 무의미하게 반복되고 있는데 제대로 매매판단을 하는건지 제공되는 로그 정보만으로는 알 수가 없어
 - 왜 저런 로그를 작성하고 있는지 관련 로직을 자세히 살펴보고, 사용자가 판단할 수 있는 정보로 표시를 해줘
