이제 차트 기능까지 구현했어. 다음으로 백테스팅 기능에 대한 명세서를 작성해줬으면 해.
앞서 작성해준 자동매매프로그램에 추가 기능이니까 기준 프로그램의 스펙은 준수해야지.

- 백테스트 모드 설정 필요?
- 기존 구현된 로직을 검증하는 역할이 가장 중요하니까 코어 엔진 변경/대체는 안돼.
  여기서 코어 엔진이라면 주로 매매 전략(전고점 돌파, 이동평균 추세 추종 등)을 말하는 거야.
  코어 엔진을 반드시 사용해서 백테스팅이 이루어져야 해.
- 현재 시점을 기준으로 KIS API 를 호출해서 계좌 및 시세 정보를 읽어오는데
  이를 준비된 백테스트 데이터로 대체해야 해. 내 생각에는 API 함수 직접 코




기존 로직을 모방해서 신규로 만드는 백테스트 모듈에 대해서 내 의견은 부정적이야.
왜냐하면 기존 로직들이 아직 안정화되지 않아서 지속 보완이 필요하고,
수정될 때마다 모방된 백테스트 모듈을 동시에 보완해야 하는 부담과 누락이나 불일치에 따른 심각한 오류도 발생할 거 같기 때문이야.
차라리 api 호출 방식을 직접 호출에서 내부함수(래퍼) 호출 방식으로 변경하는거야.
그리고 기준일자 인수를 받아서 값이 있냐 없냐에 따라 api 호출 결과를 리턴할지 백데이터 결과를 리턴할지 정하는거야.
